Styling with CSS

In 1995 a big innovation came to web authoring. Before this point much of the structure and presentation of the web was mixed together into one document. Wired Magazine was one of the first companies to use CSS to separate the content and the design. This is a vitally important step forward for web design. Now you could create all the content for a site, keep all the design separate and update your whole site much more easily.

As can be seen on CSS Zen Garden, the same HTML document can be styled an infinite number of ways with different CSS documents.

Objectives

Create an HTML file as specified and use CSS to style the contents of a page, then post it to the server.

Assignment Specifications

Use the materials provided in the assignment download module to lay out a web page about red-eyed tree frogs. There is a text file, an image, and a PDF of an example layout. The text is completely unformatted, except for the list of URLs at the end that correspond to the related links. The PDF shows the appropriate divisions of the text.

Open a new, empty HTML page and start by marking up the text in HTML. Then do all of your styling and layout using an external CSS file; NO tables, NO internal styles, NO default link styles.

You must:

  • Include separate div or section tags for the distinct sections shown in the example (header, page content, related links, footer, etc).
  • Include working links (some will be dummies, but they still need to act like links).
  • Choose styling for links, visited links, and hover links, as well as for the quoted text. Hint: the text is a longish block of quoted text.

Please note: You do not have to replicate the look of the example, only its logical structure. Different layout choices, text styling and colors are encouraged. You may substitute a different image if you like, and you may add headings to the text. Otherwise use the text as is.

Please also note: You may not use any of the layout templates provided in Dreamweaver for any of the work in this course, though you are welcome to study them for pointers. Those layouts are very handy tools you may want to use on your own, but for the purposes of this introductory course you need to learn to do your own markup and styling.
